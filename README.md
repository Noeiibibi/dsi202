# Chomchonlink

**ประเภท:** เว็บแอปพลิเคชัน  
**ภาษา:** Python (Django Framework)  
**โครงการ:** Full Stack Software Development (DSI202)

"Chomchonlink" คือแพลตฟอร์มเว็บแอปพลิเคชันที่พัฒนาขึ้นภายใต้โครงการ Full Stack Software Development (DSI202) โดยใช้ภาษา Python และ Django Framework. ด้วยแรงบันดาลใจจากแพลตฟอร์มอย่าง Meetup, Chomchonlink มุ่งหวังที่จะเสริมสร้างความผูกพันและการมีส่วนร่วมของคนในชุมชนคลองหลวง โดยเฉพาะอย่างยิ่งการเชื่อมโยงระหว่างชาวบ้านในพื้นที่และนักศึกษามหาวิทยาลัย. ระบบนี้ทำหน้าที่เป็นศูนย์รวมที่ผู้ใช้สามารถสร้าง ค้นหา และเข้าร่วมกิจกรรม กลุ่ม หรือชมรมต่างๆ ที่ตรงกับความสนใจร่วมกัน.

วัตถุประสงค์หลักของ Chomchonlink คือการส่งเสริมการสร้างความสัมพันธ์และมิตรภาพที่มีความหมายในหมู่สมาชิกชุมชน, สนับสนุนการประชาสัมพันธ์สถานที่ท่องเที่ยว กิจกรรม และประเพณีวัฒนธรรมท้องถิ่น, และช่วยให้ผู้อยู่อาศัยใหม่รวมถึงนักศึกษาสามารถปรับตัวเข้ากับชุมชนได้อย่างราบรื่นยิ่งขึ้น. ด้วยฟีเจอร์หลักที่ครอบคลุมการสร้างและเข้าร่วมกลุ่ม, การเรียกดูและค้นหากิจกรรม, เครื่องมือบริหารจัดการสำหรับผู้จัด, และระบบโปรไฟล์ผู้ใช้ที่รองรับหลากหลายบทบาท (ชาวบ้าน, นักศึกษา, ผู้จัด), Chomchonlink จึงเป็นมากกว่าแค่แพลตฟอร์ม แต่เป็นพื้นที่ที่ทุกภาคส่วนของชุมชนสามารถมาพบปะ แลกเปลี่ยน และสร้างสรรค์สิ่งดีๆ ร่วมกัน เพื่อการเติบโตของชุมชนอย่างยั่งยืน

## 3.2 User Stories (ลักษณะการใช้งาน)

Chomchonlink ถูกออกแบบมาเพื่อตอบสนองความต้องการที่หลากหลายของผู้ใช้งานในชุมชนคลองหลวง โดยมีลักษณะการใช้งานหลักดังนี้:

* **การค้นพบและเข้าร่วมกิจกรรมสำหรับสมาชิกใหม่/นักศึกษา:**
    "ในฐานะนักศึกษามหาวิทยาลัยใหม่, ฉันต้องการเข้าร่วมกิจกรรมในท้องถิ่นเพื่อที่ฉันจะได้พบปะผู้คนใหม่ๆ และรู้สึกผูกพันกับพื้นที่นี้มากขึ้น"

* **การมีส่วนร่วมในวัฒนธรรมและประเพณีสำหรับชาวบ้าน:**
    "ในฐานะชาวบ้านในพื้นที่, ฉันต้องการค้นหากิจกรรมทางวัฒนธรรมและเทศกาลต่างๆ เพื่อที่ฉันจะได้มีส่วนร่วมในประเพณีของชุมชนของฉัน"

* **การค้นหาโอกาสช่วยเหลือสังคมและการมีส่วนร่วมสำหรับผู้สูงอายุ/ผู้ที่ต้องการทำประโยชน์:**
    "ในฐานะผู้เกษียณ, ฉันต้องการค้นพบโอกาสในการเป็นอาสาสมัครเพื่อที่จะได้กระตือรือร้นและมีส่วนร่วม"

* **การประชาสัมพันธ์กิจกรรมสำหรับผู้ประกอบการ/ผู้จัด:**
    "ในฐานะเจ้าของธุรกิจในท้องถิ่น, ฉันต้องการประชาสัมพันธ์กิจกรรมของฉันเพื่อที่ฉันจะได้เข้าถึงผู้คนในละแวกใกล้เคียงได้มากขึ้น"

* **การเชื่อมโยงกับผู้ที่มีความสนใจคล้ายกัน:**
    "ในฐานะคนที่ชื่นชอบกิจกรรมกลางแจ้ง, ฉันต้องการเชื่อมต่อกับผู้คนที่มีงานอดิเรกคล้ายกัน"

## 3.3 ขั้นตอนการใช้งานตาม User Stories

เพื่อให้เห็นภาพการใช้งาน Chomchonlink ที่ชัดเจนขึ้น เราจะอธิบายขั้นตอนการใช้งานสำหรับ User Stories หลักที่กำหนดไว้:

### User Story 1: ในฐานะนักศึกษามหาวิทยาลัยใหม่, ฉันต้องการเข้าร่วมกิจกรรมในท้องถิ่นเพื่อที่ฉันจะได้พบปะผู้คนใหม่ๆ และรู้สึกผูกพันกับพื้นที่นี้มากขึ้น

**ขั้นตอนที่ 1: การสมัครสมาชิก/เข้าสู่ระบบ**
* นักศึกษาเข้าสู่หน้าแรกของ Chomchonlink.
* คลิกที่ปุ่ม "สมัครเลย!" เพื่อสร้างบัญชีผู้ใช้ใหม่ หรือ "เข้าสู่ระบบ" หากมีบัญชีอยู่แล้ว.
* หากสมัครใหม่ กรอกข้อมูลที่จำเป็น (ชื่อผู้ใช้, อีเมล, รหัสผ่าน) หรือเลือก "สมัครสมาชิกด้วย Google" เพื่อความสะดวก.
* ระบบจะนำไปยังหน้าแรกพร้อมแสดงข้อความต้อนรับ.

**ขั้นตอนที่ 2: การค้นหากิจกรรม**
* เมื่อเข้าสู่ระบบแล้ว, นักศึกษาคลิกที่เมนู "กิจกรรม" เพื่อดูรายการกิจกรรมทั้งหมด.
* นักศึกษาสามารถใช้ช่องค้นหาเพื่อพิมพ์คำที่สนใจ เช่น "พิมพ์ชื่อกิจกรรม, สถานที่, หรือคำค้นหา...".
* นอกจากนี้, ยังสามารถใช้ตัวกรอง "ชุมชน" เพื่อเลือกดูเฉพาะกิจกรรมที่จัดในชุมชนที่สนใจ หรือ "หมวดหมู่" เพื่อหากิจกรรมตามประเภท.

**ขั้นตอนที่ 3: การสำรวจและแสดงความสนใจ/เข้าร่วม**
* นักศึกษาเลื่อนดูรายการกิจกรรมที่แสดงผล.
* เมื่อพบกิจกรรมที่น่าสนใจ, คลิกที่รูปภาพหรือชื่อกิจกรรมเพื่อดูรายละเอียดเพิ่มเติม.
* ในหน้ากิจกรรม, นักศึกษาสามารถอ่านคำอธิบาย, วันที่, เวลา, สถานที่จัด.
* เลือกปุ่ม "ยืนยันเข้าร่วม" หากต้องการเข้าร่วมจริง หรือ "สนใจกิจกรรมนี้" หากต้องการบันทึกไว้ดูภายหลัง.
* ระบบจะแจ้งเตือนว่าสถานะการเข้าร่วม/ความสนใจได้รับการบันทึกเรียบร้อย.

**ขั้นตอนที่ 4: การจัดการโปรไฟล์และการเชื่อมต่อ**
* นักศึกษาสามารถคลิกที่ "โปรไฟล์" เพื่อดูประวัติกิจกรรมที่เข้าร่วมหรือสนใจ.
* สามารถอัปเดตข้อมูล Bio หรือรูปโปรไฟล์เพื่อแนะนำตัวเองให้ผู้อื่นรู้จัก.

### User Story 2: ในฐานะเจ้าของธุรกิจในท้องถิ่น, ฉันต้องการประชาสัมพันธ์กิจกรรมของฉันเพื่อที่ฉันจะได้เข้าถึงผู้คนในละแวกใกล้เคียงได้มากขึ้น

**ขั้นตอนที่ 1: การสมัครสมาชิก/เข้าสู่ระบบ**
* เจ้าของธุรกิจเข้าสู่หน้าแรกของ Chomchonlink.
* ทำการสมัครสมาชิกหรือเข้าสู่ระบบในลักษณะเดียวกับนักศึกษา (ข้างต้น).

**ขั้นตอนที่ 2: การสร้างกิจกรรมใหม่**
* เมื่อเข้าสู่ระบบแล้ว, คลิกที่เมนู "สร้างกิจกรรม" หรือปุ่มที่เกี่ยวข้อง.
* กรอกรายละเอียดกิจกรรมในฟอร์มที่กำหนด: ชื่อกิจกรรม, คำอธิบาย, สถานที่, วันที่, เวลา, รูปภาพ, หมวดหมู่ และชุมชนที่เกี่ยวข้อง.
* คลิก "สร้างกิจกรรมเลย!" เพื่อส่งข้อมูล.

**ขั้นตอนที่ 3: การจัดการกิจกรรม**
* ระบบจะนำไปยังหน้ากิจกรรมที่สร้างขึ้น และแสดงข้อความยืนยันการสร้างกิจกรรมพร้อมแต้มที่ได้รับ.
* เจ้าของธุรกิจสามารถกลับไปที่หน้าโปรไฟล์ของตนเอง ("โปรไฟล์") เพื่อดูรายการ "กิจกรรมที่จัด" ทั้งหมด.
* จากหน้ารายการกิจกรรมที่จัด, สามารถคลิก "แก้ไขกิจกรรม" เพื่อปรับปรุงข้อมูลกิจกรรมได้ตลอดเวลา.

**ขั้นตอนที่ 4: การตรวจสอบผู้เข้าร่วม**
* ในหน้ากิจกรรมของตนเอง, เจ้าของธุรกิจสามารถดูรายชื่อ "ผู้เข้าร่วม" และ "ผู้ที่สนใจ" เพื่อประเมินจำนวนผู้ที่คาดว่าจะมาเข้าร่วม.

## 3.4 การติดตั้งและใช้งาน (Local Development พร้อม Docker)

สำหรับการติดตั้งและรันโปรเจกต์ Chomchonlink บนเครื่องของคุณด้วย Docker, ให้ทำตามขั้นตอนดังนี้:

**ความต้องการเบื้องต้น:**
* Docker Desktop (รวม Docker Engine และ Docker Compose) ติดตั้งบนเครื่องของคุณ
* Git (สำหรับการโคลน repository)

**ขั้นตอนการติดตั้งและใช้งาน (วิธีที่แนะนำ: Docker Compose):**

1.  **โคลน Repository:**
    เปิด Terminal หรือ Command Prompt แล้วโคลนโปรเจกต์จาก GitHub (สมมติว่าคุณมี URL ของ Git Repository):
    ```bash
    git clone <your_repository_url>
    cd dsi202_2025-01912189329c2f310424833d7e680df76231513b # ตัวอย่างชื่อโฟลเดอร์จากไฟล์ที่ให้มา
    ```

2.  **ตั้งค่าไฟล์ `.env`:**
    หากโปรเจกต์มีการใช้ Environment Variables (เช่น `DJANGO_SECRET_KEY`, `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`) คุณอาจจะต้องสร้างไฟล์ `.env` ที่ root ของโปรเจกต์ (ใน directory เดียวกับ `docker-compose.yml`) และกำหนดค่าเหล่านี้. จากไฟล์ `docker-compose.yml` ที่ให้มา, คุณจะต้องกำหนด `GOOGLE_CLIENT_ID` และ `GOOGLE_CLIENT_SECRET`. ตัวอย่าง `.env` ไฟล์:
    ```
    DJANGO_SECRET_KEY='your_very_secret_key_here'
    GOOGLE_CLIENT_ID='YOUR_ACTUAL_CLIENT_ID_FROM_GOOGLE_HERE'
    GOOGLE_CLIENT_SECRET='YOUR_ACTUAL_CLIENT_SECRET_FROM_GOOGLE_HERE'
    ```
    *แทนที่ค่า `your_very_secret_key_here`, `YOUR_ACTUAL_CLIENT_ID_FROM_GOOGLE_HERE`, และ `YOUR_ACTUAL_CLIENT_SECRET_FROM_GOOGLE_HERE` ด้วยค่าจริงของคุณ*.

3.  **สร้างและรัน Docker Containers:**
    จาก root directory ของโปรเจกต์ (ที่เดียวกับ `docker-compose.yml`), รันคำสั่ง Docker Compose:
    ```bash
    docker-compose up --build -d
    ```
    * `up`: สร้างและเริ่มต้น services ที่ระบุใน `docker-compose.yml`.
    * `--build`: บังคับให้ Docker สร้าง image ใหม่จาก `Dockerfile.django` และ `Dockerfile.jupyter` (หากมีการเปลี่ยนแปลง Dockerfile หรือ dependencies).
    * `-d`: รัน containers ในโหมด "detached" (ทำงานอยู่เบื้องหลัง).

4.  **ตรวจสอบสถานะ Containers:**
    คุณสามารถตรวจสอบว่า containers ทำงานอยู่หรือไม่โดยใช้:
    ```bash
    docker-compose ps
    ```
    คุณควรเห็น `postgres`, `django`, และ `jupyter` services รันอยู่.

5.  **เข้าถึง Web Application:**
    เมื่อ containers รันเสร็จสิ้น, คุณสามารถเข้าถึง Chomchonlink Web Application ได้ที่:
    `http://127.0.0.1:8000`.

6.  **เข้าถึง Jupyter Notebook (ถ้าต้องการ):**
    หากคุณต้องการใช้ Jupyter Notebook สำหรับการสำรวจข้อมูลหรือพัฒนาเพิ่มเติม, สามารถเข้าถึงได้ที่:
    `http://127.0.0.1:8888`.
    *คุณอาจจะต้องใช้ Token ที่แสดงใน Terminal ที่รัน `docker-compose up` ครั้งแรก เพื่อเข้าสู่ Jupyter*.

7.  **หยุดและลบ Docker Containers (เมื่อเสร็จสิ้นการใช้งาน):**
    เมื่อคุณใช้งานเสร็จสิ้นและต้องการหยุด containers และลบ network/volumes (ยกเว้น `db_data` volume ที่เก็บข้อมูลฐานข้อมูล):
    ```bash
    docker-compose down
    ```
    หากต้องการลบ volumes ทั้งหมด (รวมถึงข้อมูลฐานข้อมูล):
    ```bash
    docker-compose down --volumes
    ```

**การแก้ไขปัญหาเบื้องต้น (Troubleshooting):**

* **"Container failed to start" / "Error connecting to database":**
    * ตรวจสอบว่า `postgres` container รันอยู่. บางครั้งฐานข้อมูลอาจต้องใช้เวลาเล็กน้อยในการเริ่มต้นก่อนที่ Django web app จะเชื่อมต่อได้ หาก `django` container restart บ่อยๆ, ลองรัน `docker-compose up` อีกครั้ง หรือเพิ่ม `healthcheck` ใน `docker-compose.yml` สำหรับ `db` service.
    * ตรวจสอบ Environment Variables สำหรับฐานข้อมูลใน `docker-compose.yml` ของ `web` service ว่าตรงกับ `db` service หรือไม่ (เช่น `POSTGRES_DB`, `POSTGRES_USER`, `POSTGRES_PASSWORD`, `POSTGRES_HOST`).

* **"manage.py not found" / "Permissions denied":**
    * ตรวจสอบว่าไฟล์ `entrypoint.sh` มีสิทธิ์ execute (`chmod +x entrypoint.sh`) และ path ใน `docker-compose.yml` สำหรับ `command` ของ `web` service นั้นถูกต้อง.
    * ตรวจสอบว่า `volumes` ใน `docker-compose.yml` ถูกต้อง (`.:/usr/src/app:rw`) เพื่อให้โค้ดจาก host ถูก mount เข้าไปใน container อย่างถูกต้อง.

* **"Dependencies not installed":**
    * ตรวจสอบ `requirements_django.txt` และ `Dockerfile.django` ว่ามีการติดตั้ง dependencies ทั้งหมดอย่างถูกต้อง.
    * หากมีการเปลี่ยนแปลง `requirements.txt` ให้รัน `docker-compose build web` ก่อน `docker-compose up`.

## 4. ระบบสามารถทำงานได้ไม่เกิดปัญหา: `docker-compose up` และการทำงานตามที่กำหนด

จากการวิเคราะห์ไฟล์ GitHub ที่ให้มา, ระบบ Chomchonlink ถูกออกแบบมาให้สามารถทำงานได้อย่างราบรื่นผ่าน Docker Compose โดยมีส่วนประกอบหลักดังนี้:

* **ฐานข้อมูล PostgreSQL:** `docker-compose.yml` กำหนด `db` service โดยใช้ `postgres:14` image พร้อมการ persist ข้อมูลผ่าน `db_data` volume และกำหนด environment variables สำหรับการเชื่อมต่อฐานข้อมูล (`POSTGRES_DB`, `POSTGRES_USER`, `POSTGRES_PASSWORD`). การตั้งค่านี้ช่วยให้ฐานข้อมูลมีความเสถียรและข้อมูลไม่หายไปเมื่อ container ถูกหยุดชั่วคราว.

* **Web Application (Django):** `web` service ถูกสร้างจาก `Dockerfile.django` (ซึ่งเป็น Dockerfile ที่อยู่ใน `context: .`). มีการ mount volume (`.:/usr/src/app:rw`) เพื่อให้โค้ดของโปรเจกต์บน host สามารถเข้าถึงได้ภายใน container, ทำให้การพัฒนาสะดวกยิ่งขึ้นโดยไม่ต้อง build image ใหม่ทุกครั้งที่แก้โค้ด.
    * `entrypoint.sh` มีบทบาทสำคัญในการตรวจสอบว่าโปรเจกต์ Django ถูกสร้างขึ้นหรือไม่ (ถ้ายังไม่มี `manage.py` ใน `/usr/src/app/myproject`) รัน `migrate` เพื่ออัปเดตฐานข้อมูล และเริ่มต้น Django development server (`runserver 0.0.0.0:8000`).
    * พอร์ต 8000 ของ container ถูก expose ออกมาที่ host machine (`8000:8000`) ทำให้สามารถเข้าถึงเว็บแอปพลิเคชันได้.
    * Environment variables ที่จำเป็นสำหรับการเชื่อมต่อกับฐานข้อมูลและ Google OAuth (`GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`) ถูกส่งผ่านจาก `docker-compose.yml`.

* **Jupyter Notebook:** `jupyter` service ถูกสร้างจาก `Dockerfile.jupyter` (ซึ่งเป็น Dockerfile ที่อยู่ใน `context: .`). มี `working_dir` ที่ชี้ไปยัง `/usr/src/app/myproject` และ mount volume เดียวกันกับ `web` service, ทำให้สามารถเข้าถึงโค้ดและข้อมูลในโปรเจกต์ Django ได้จาก Jupyter.
    * พอร์ต 8888 ถูก expose สำหรับการเข้าถึง Jupyter interface.

**ความสามารถในการทำงานและประสิทธิภาพตามที่กำหนด:**

* **การตั้งค่าเริ่มต้นอัตโนมัติ:** `entrypoint.sh` ช่วยให้มั่นใจว่าฐานข้อมูลจะถูก migrate โดยอัตโนมัติเมื่อ `web` container เริ่มต้น, ลดขั้นตอนการตั้งค่าด้วยมือ.
* **การจัดการแต้มผู้ใช้:** ระบบมีการจัดการแต้มผู้ใช้ที่ชัดเจนผ่าน `UserProfile` และ `PointActivityLog` models. ฟังก์ชัน `manage_points` ถูกแยกออกมาเป็น utility function (`myapp/utils.py`) และใช้ใน `signals.py` เพื่อมอบแต้มเมื่อผู้ใช้สมัครสมาชิก/เข้าสู่ระบบ. นอกจากนี้ยังมีการมอบแต้มเมื่อผู้ใช้สร้างกิจกรรม (25 แต้ม) หรือเข้าร่วมกิจกรรม (10 แต้ม).
* **ระบบรางวัล:** มี `Reward` model ที่รองรับประเภทรางวัลต่างๆ เช่น Badge, Profile Frame, Discount Code, Digital Good. ผู้ใช้สามารถแลกรางวัลได้ผ่าน `rewards_store_view` โดยมีการตรวจสอบแต้มที่เพียงพอและป้องกันการแลก Badge ซ้ำ.
* **โปรไฟล์ผู้ใช้:** ผู้ใช้สามารถดูโปรไฟล์ของตนเองและผู้อื่นได้. มีหน้าสำหรับแก้ไขโปรไฟล์ (Bio, รูปโปรไฟล์) และเลือกใช้ Profile Frame ที่แลกมาได้.
* **การจัดการกิจกรรมและ Attendance:** ผู้ใช้สามารถสร้าง (`EventCreateView`), ดูรายละเอียด (`EventDetailView`), แก้ไข (สำหรับผู้จัด `EventUpdateView`), และเข้าร่วมกิจกรรม (`attend_event`) ได้. มีระบบติดตามสถานะการเข้าร่วม (`attending`, `interested`, `not_attending`).
* **Social Authentication:** การรวม `django-allauth` และ `allauth.socialaccount.providers.google` (กำหนดใน `settings.py`) ทำให้ผู้ใช้สามารถเข้าสู่ระบบและสมัครสมาชิกผ่าน Google ได้, เพิ่มความสะดวกสบายในการใช้งาน.
* **Dashboard สำหรับ Admin:** มีหน้า Dashboard พื้นฐานสำหรับผู้ดูแลระบบเพื่อดูสถิติจำนวนผู้ใช้, กิจกรรม, ชุมชน, และการเข้าร่วม.

โดยรวมแล้ว, การกำหนดค่าใน `docker-compose.yml` และการจัดโครงสร้างโค้ดของ Django บ่งชี้ว่าระบบถูกออกแบบมาให้สามารถ `docker-compose up` ได้โดยไม่มีปัญหาในการเริ่มต้นและสามารถทำงานตามฟังก์ชันที่ระบุไว้ได้อย่างครบถ้วน. หากมีการตั้งค่า `GOOGLE_CLIENT_ID` และ `GOOGLE_CLIENT_SECRET` อย่างถูกต้อง, ฟังก์ชันการทำงานของ Google Social Login ก็ควรจะทำงานได้อย่างราบรื่นเช่นกัน.


"Chomchonlink" ไม่เพียงเป็นแพลตฟอร์มดิจิทัลที่เชื่อมโยงผู้คนเข้าหากันเท่านั้น แต่ยังเป็นหัวใจสำคัญในการขับเคลื่อนการมีส่วนร่วมทางสังคมและวัฒนธรรมในพื้นที่คลองหลวง. ในยุคที่สังคมมักแยกห่างกันด้วยหน้าจอ, Chomchonlink นำเสนอทางออกที่สร้างสรรค์ ด้วยการเป็นสะพานเชื่อมโยง "เพื่อนบ้าน" ที่แท้จริง ไม่ว่าจะเป็นนักศึกษาผู้มาใหม่ที่กำลังมองหาสถานที่สำหรับ "ผีเสื้อโบยบิน" สร้างมิตรภาพใหม่ๆ หรือชาวบ้านที่ต้องการสืบสานประเพณีและแบ่งปันความสุข. แพลตฟอร์มนี้ไม่เพียงอำนวยความสะดวกในการค้นหากิจกรรมที่ตอบโจทย์ทุกความสนใจ แต่ยังส่งเสริมให้ทุกคนเป็นส่วนหนึ่งของชุมชนที่อบอุ่นและมีชีวิตชีวา. ด้วยระบบแต้มและรางวัลที่กระตุ้นการมีส่วนร่วม, รวมถึงฟังก์ชันการสร้างกิจกรรมที่ง่ายดาย, Chomchonlink จึงเป็นเครื่องมือทรงพลังที่ช่วยให้ผู้ประกอบการท้องถิ่นสามารถประชาสัมพันธ์ธุรกิจและกิจกรรมของตนได้อย่างมีประสิทธิภาพ, และช่วยให้ผู้สูงอายุสามารถค้นพบโอกาสในการเป็นอาสาสมัครเพื่อสร้างคุณค่าให้แก่สังคม. นี่คือการลงทุนในอนาคตของชุมชน ที่ซึ่งทุกการคลิกนำไปสู่การเชื่อมต่อที่แท้จริง และทุกกิจกรรมร่วมกันสร้างความทรงจำอันล้ำค่า อันเป็นประโยชน์อย่างยิ่งในการสร้างชุมชนที่เข้มแข็งและยั่งยืน
